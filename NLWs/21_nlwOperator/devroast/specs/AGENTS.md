# Diretrizes de Specs

Todas as novas funcionalidades ou refatorações complexas devem começar com um arquivo de especificação no diretório `specs/`.

## Formato
Cada spec deve ser um arquivo `.md` contendo:

1. **Visão Geral**: Descrição breve do objetivo.
2. **Requisitos**: Lista de requisitos funcionais e não funcionais.
3. **Abordagem Técnica**: Detalhes de implementação, mudanças arquiteturais e ferramentas a serem utilizadas.
4. **Verificação**: Como testar e verificar a implementação.

## Processo
1. Criar o arquivo de spec.
2. Revisar e refinar os requisitos.
3. Implementar a funcionalidade com base na spec.
4. Marcar a spec como concluída após a verificação.
